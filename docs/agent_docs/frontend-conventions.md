# 프론트엔드 코딩 컨벤션

> UI 컴포넌트 구현, 스타일링, 반응형 작업 시 참조

## 스타일링

- Tailwind CSS 유틸리티 클래스 사용
- 커스텀 CSS 변수: `iv-` 접두사 (globals.css 참조)
- shadcn/ui 컴포넌트 추가: `pnpm dlx shadcn@latest add <component>`
- cn() 유틸리티로 클래스 병합 (@/lib/utils)

## 반응형 & 모바일 UX

**브레이크포인트 (Mobile-First)**

| 접두사 | 최소 너비 | 대상 디바이스 |
| ------ | --------- | ------------- |
| (없음) | 0px       | 모바일 기본   |
| `sm`   | 640px     | 대형 모바일   |
| `md`   | 768px     | 태블릿        |
| `lg`   | 1024px    | 데스크톱      |

- **Mobile-First 필수**: 기본 스타일은 모바일 → `sm:`, `md:`, `lg:`로 확장
- 데스크톱 스타일을 기본으로 작성하고 모바일을 덮어쓰는 방식 금지

**터치 & 가독성**

- 터치 타겟 최소 44×44px (버튼, 링크, 인터랙티브 요소)
- 모바일 본문 텍스트 최소 `text-sm` (14px), 가능하면 `text-base` (16px)
- input `font-size` 16px 이상 유지 (iOS 자동 확대 방지)
- 긴 텍스트는 `break-words` 또는 `truncate` 처리

**레이아웃 패턴**

- 카드/리스트: 모바일 1열 → `md:` 이상에서 2열 이상 그리드
- 사이드바: 모바일에서 숨김 또는 Sheet/Drawer로 전환
- 모달/다이얼로그: 모바일에서 `DrawerDialog` 패턴 사용 (shadcn Drawer + Dialog 조합)
- 테이블: 모바일에서 카드형 레이아웃으로 전환 또는 가로 스크롤 허용
- 간격(padding/margin): 모바일 `p-4` → 데스크톱 `md:p-6` 이상으로 점진 확대

**구현 규칙**

- 고정 너비(`w-[500px]`) 사용 지양 → `max-w-*`, `w-full` 등 유연한 단위 사용
- `overflow-hidden` 남용 금지 — 콘텐츠 잘림 여부 반드시 확인
- 새 페이지/컴포넌트 구현 시 모바일(375px) 뷰포트에서 반드시 확인
- 기존 컴포넌트 수정 시에도 모바일 깨짐 여부 체크
